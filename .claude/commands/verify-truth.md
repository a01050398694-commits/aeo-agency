# 직전 작업 진실성 검증

## 사용 시점
- 작업 완료 보고 직후 의심될 때
- 사용자가 "진짜 했어?" 묻기 전에 자가 검증
- 외부에 보고하기 직전 최종 검수

## 실행

`truth-verifier` 서브에이전트 호출:
- 직전 N개 메시지의 모든 주장 추출
- 각 주장에 대해 검증 수행
- 거짓/미완료 발견 시 즉시 재작업

## 보고 형식

```
🔍 검증 결과

✅ 검증 통과 (X건):
- ...

❌ 거짓/미완료 (Y건):
- ...

📌 권고:
- ...
```

## 절대 규칙
- 검증 결과를 호의적으로 해석 금지
- 의심스러우면 "검증 안 됨" 처리
- 거짓 발견 시 사용자에게 즉시 통보
