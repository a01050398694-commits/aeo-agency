# Client Onboarding

## 수집 정보 체크리스트

### A. 비즈니스 기본 (필수)
- [ ] 정식 법인명 (한글 + 영문)
- [ ] 브랜드명
- [ ] 도메인
- [ ] NAP (주소/전화/이메일)
- [ ] 창업일 (YYYY-MM-DD)
- [ ] 대표/창업자 풀네임
- [ ] 사업자등록번호
- [ ] 회사 설명 (50자/150자/500자)
- [ ] 로고 (정사각 512px+ PNG)
- [ ] 주요 카테고리
- [ ] 서비스 지역

### B. 디지털 자산 권한 (필수)
- [ ] CMS 종류 + API 토큰
- [ ] Google Search Console 권한
- [ ] Google Analytics 4 권한
- [ ] Google Business Profile 매니저
- [ ] YouTube 채널 매니저 (있을 시)
- [ ] Bing Webmaster Tools
- [ ] Gemini API 키 (자체 또는 우리 키 사용)

### C. 콘텐츠 자산
- [ ] 기존 페이지 URL 리스트
- [ ] 제품/서비스 페이지
- [ ] 백서/연구 자료
- [ ] 언론 보도 링크
- [ ] YouTube 영상
- [ ] 이미지 라이브러리

### D. 전략 가드레일
- [ ] 타깃 쿼리 50~100개
- [ ] 경쟁사 5~10곳 도메인
- [ ] 브랜드 보이스 가이드
- [ ] 금지 주장 리스트
- [ ] 승인된 인증/수상/통계
- [ ] 법무 연락처

### E. 엔티티 신호
- [ ] LinkedIn 회사 페이지
- [ ] Crunchbase
- [ ] 소셜 채널 (X/IG/FB/YT)
- [ ] 업종 디렉토리

## 실행 절차

1. **사용자에게 정보 수집 양식 한 번에 제시**
   - 위 체크리스트를 markdown 양식으로 사용자에게 전달
   - 누락 항목은 표시
2. **클라이언트 슬러그 결정** (예: `acme-corp`)
3. **디렉토리 생성**: `clients/<slug>/`
4. **profile.yaml 작성**
5. **.env 파일 생성** (자격증명, .gitignore 적용)
6. **초기 진단 실행** (베이스라인 측정)
7. **로드맵 생성**: 30/60/90일

## 절대 금지
- ❌ 정보 누락된 채로 진단 강행
- ❌ 자격증명을 profile.yaml에 저장 (반드시 .env)
- ❌ 다른 클라이언트와 정보 혼합
- ✅ 모든 필드 검증 (URL 200, 이메일 형식 등)
